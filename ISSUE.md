## Q2: "反 AI 味"的定义从哪来？

8 个维度——Inter 字体、紫色渐变、毛玻璃、统一圆角——是 2023-2024 年 AI 生成 UI 的特征。但 AI 生成工具在快速迭代：

- GPT-4o 生成的 UI 已经不怎么用 Inter 了

## 附录 A：根因诊断（2026-06-28 会话）

> 基于 src/scorer/eight_dim.py、src/generator/llm_generator.py、catalog/SEEDS.md、
> styles/bauhaus-berlin、styles/bauhaus-chicago 等样本的代码与产物审查。

### A1. 根本病因：把"分类学"误当成"产品"

`catalog/SEEDS.md` 的 14,639 个风格全部来自 `Tier1 × 地域 × 媒介 × 色盘 × 材质` 的**机械笛卡尔积**：

- `bauhaus-berlin` 的 anti_ai_signal = "形式追随功能；原色+黑白；几何构成,**柏林特征**"
- `bauhaus-chicago` 的 anti_ai_signal = "形式追随功能；原色+黑白；几何构成,**芝加哥特征**"
- `bauhaus-tel-aviv`、`bauhaus-moscow`… 同上，只换地名

真正的设计运动约 50 个（Tier 1），乘以 293 种排列组合，制造了"15,000 个风格"的幻象。
**目录的规模和它的信息量成反比**：14,639 不是 14,639 个设计规范，而是 50 个规范 × 293 份近似副本。

### A2. 评分器：循环论证的闭环

`eight_dim.py` 是 8 条 regex。`llm_generator.py` 的 `SYSTEM_PROMPT` 直接把"不要做 X、Y、Z"喂给 LLM，
然后评分器检查 X、Y、Z 是否出现。retry 反馈（`llm_generator.py:228-237`）把失败维度直接回灌给 LLM，
等于在告诉 LLM **如何绕过评分器**。

评分器验证的不是"设计是否反 AI 味"，而是"LLM 是否服从了我写的 8 条 CSS 禁令"。

### A3. 生成质量：占位符伪装成规范

对比 `bauhaus-berlin` 与 `bauhaus-chicago` 的 `DESIGN.md`：

| 字段 | bauhaus-berlin | bauhaus-chicago |
|---|---|---|
| primary | `#E3000F` Bauhaus Red | `#E3000F` |
| secondary | `#003898` Bauhaus Blue | `#0047AB` |
| tertiary | `#FFCC00` Bauhaus Yellow | `#FFD100` |
| h1 font | Futura, Century Gothic | Akzidenz-Grotesk |

色值几乎一样，字体都是 LLM 从"Bauhaus 训练分布"里采样的常见答案。
"柏林特征"和"芝加哥特征"在产物里完全不可见——Berlin 的 description 是
"industrial functionalism of the Weimar era"，和芝加哥毫无关系。
14,623 个 HTML 文件，**真正可区分的设计语言估计 ≤ 30 种**，其余 14,600 个是噪声。

### A4. 实测数据

- styles 目录总数：14,639
- 有 reference.html 的：14,623
- "validated" 通过率：14,597/14,639 = **99.7%**——这不是质量信号，是评分器被绕过的成功率

## 附录 B：囚徒困境的本质

每一方都在做"看起来对"的事，但合起来得到最差的结果——典型的"局部优化导致全局劣化"。

| 角色 | 局部最优 | 全局后果 |
|---|---|---|
| 评分器 | 多加一条 regex 提高"覆盖率" | 把 LLM 训练成绕规则专家 |
| 生成器 | 多重试几次让分数 ≤4 | 14K 个"达标"但雷同的 HTML |
| 种子库 | 多乘一个维度凑规模 | 笛卡尔积稀释真实多样性 |
| Gallery | 展示数量 = "资产" | 99.9% 是 404，暴露 demo 性质 |

**没有人背叛，但所有人都被困住了。** 每个模块都在自己的 KPI 里勤勉工作，KPI 本身错了。

## 附录 D：客观判断——是否还有做下去的必要

### 默认结论：当前形态不值得继续

四个维度同时失效，且没有渐进改良路径。继续加维度、加风格变体只会把囚徒困境放大。

数据已经回答了所有关键问题：

- 评分器有科学性吗？——8 条 regex，零人类标注验证
- 14,623 个 HTML 有差异性吗？——bauhaus-berlin vs bauhaus-chicago 色值几乎相同，"柏林特征"不可见
- 通过率高 = 质量高吗？——99.7% 是评分器被绕过的证据，不是质量证据
- 有真实用户吗？——Gallery 仅 10 个有 HTML，其余 404，不是产品
- 有护城河吗？——OpenAI 一个 CSS lint 插件即可替代

### 常见的自欺叙事（均不成立）

- ❌ "再迭代几版评分器" → 没有人类标注，再迭代 100 版还是循环论证
- ❌ "再加几个维度" → 笛卡尔积再加 3 倍还是笛卡尔积
- ❌ "AI 风格在变，我们跟着更新" → 负毛利业务，规模越大越亏
- ❌ "积累了 14K 个资产" → 14K 个 LLM 平均回声是负债，不是资产
- ❌ "已经投入了这么多" → 沉没成本是停止的理由，不是继续的理由

### 唯一值得做的重构方案

**如果**愿意做以下四件事，**那么**项目有重生可能：

1. **14,639 → 50**：删除所有 Tier 2-6 笛卡尔积变体，只保留 Tier 1 真实运动，每个做精（真实色板、字体溯源、历史参考文献）。
2. **8 条 regex → 200 张人类标注 + ML 分类器**（哪怕 logistic regression）。没有这一步，所有"validated"都是空中楼阁。
3. **盲测验证**：5-10 个设计师盲评生成 HTML vs 真实设计稿，证明评分器与人类判断相关（precision/recall/IoU）。这是 Q4、Q5 唯一的解药。
4. **定位二选一**：
   - 选项 A："反 AI 味 CSS lint 规则包"（npm 包，CI 集成）
   - 选项 B："AI 生成 UI 检测器"（学术 + 工具，对标 GPTZero）

   现在的"生成器+自评器"循环不是定位，是身份混乱。

### 必须停止做的事

- ❌ 继续生成更多风格变体
- ❌ 继续扩展 Tier 7、Tier 8（YAGNI）
- ❌ 继续优化 retry 反馈机制——这是在教 LLM 绕过评分器
- ❌ 继续往 Gallery 加 404 链接

### 最终判断

> 当前形态：**不值得继续**——四个维度同时失效，且没有渐进改良路径。
> 重构形态：**值得继续**，但前提是砍掉 90% 产出 + 加人类验证 + 定位二选一。

**承认方向错误比坚持错误更便宜。** 沉没成本已经付了，未来的时间还没付——后者更值钱。
重构后的"反 AI 味 CSS lint 包"或"AI UI 检测器"与现在的 14K 风格库没有继承关系——
14K 风格库应归档，作为"我们试过这条路、发现是死路"的负样本。

## Q1: 核心贡献是什么？

8 维评分器本质上是 8 条正则表达式。`font_cliche` 正则匹配 font-family、`purple_gradient` 匹配十六进制色值、`element_density` 数 HTML 标签数——都是 if-else + regex。没有学习、没有泛化、没有语义理解。

**追问:** 一个前端工程师花 2 小时就能写出来。"scorer"和一个 lint rule 有什么本质区别？凭什么这是一个"项目"而不是一个"脚本"？

## Q2: "反 AI 味"的定义从哪来？

8 个维度——Inter 字体、紫色渐变、毛玻璃、统一圆角——是 2023-2024 年 AI 生成 UI 的特征。但 AI 生成工具在快速迭代：

- GPT-4o 生成的 UI 已经不怎么用 Inter 了
- Claude 生成的 UI 风格每几个月变一次
- 新的 AI 工具（v0, Bolt, Lovable）各有自己的风格指纹

**追问:** rubric 是静态的。6 个月后这 8 条规则还有多少覆盖率？怎么证明这些维度是"AI 味"而不是"2024 年流行的 UI 趋势"？

**回应:** 待补充。

## Q3: 55 个设计运动的种子库——知识还是数据？

Tier 2-6 本质上是 Tier 1 × 地域 × 媒介 × 色盘 × 材质的笛卡尔积。`bauhaus-berlin`、`bauhaus-chicago`、`bauhaus-tel-aviv` 的 DNA 字段几乎一模一样，只是加了"柏林特征"、"芝加哥特征"这种占位符。

**追问:** 15,000 个有意义的设计规范，还是 55 个 × 270 种排列组合？笛卡尔积能算"贡献"吗？

## Q4: LLM 生成 + 评分门控 = 闭环验证？还是循环论证？

Pipeline 是 LLM 生成 HTML → regex 评分 → 不过就重试。评分器是自己写的规则，prompt 也是自己写的。

**追问:** 验证的是"LLM 能不能遵守你写的 8 条 CSS 规则"，而不是"这个设计是否真的不像 AI"。真正的验证需要人类评估：盲测、precision/recall、和评分器的相关性分析。做过吗？

**回应:** 待补充。

## Q5: 假阳性和假阴性率是多少？

- **假阳性:** 一个用 Inter 字体的设计师手工作品会被打 2 分
- **假阴性:** 一个用 Comic Sans 的 AI 生成页面完全不触发规则，得 0 分

**追问:** 能量化误差率吗？惩罚所有使用现代流行字体的设计（不管是不是 AI），对不在 8 条规则内的 AI 模式完全无感。
